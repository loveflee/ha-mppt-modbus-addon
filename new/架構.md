邊緣閘道器（Edge Gateway）架構所規劃的模組清單與資料流 SOP。所有設計皆圍繞「總線單一控制權」、「快慢車道分離」與「設備配置解耦」的底層邏輯進行拆解。
一、 核心模組清單與職責劃分
系統建議拆分為 5 個核心模組與 1 個配置中心。模組之間應透過明確的介面（或非同步佇列）溝通，避免互相綁死。
1. 配置與狀態中心 (Config & State Registry)
 * 職責：系統啟動時的唯一真理來源（Single Source of Truth），並在運行時維護設備狀態。
 * 重點要求：
   * 解析 config.yaml（通訊參數、設備 UID、輪詢間隔）。
   * 載入並快取各設備的「點位地圖檔 (Map)」。
   * 維護運行期狀態記憶體：記錄每個 UID 的當前狀態（ONLINE / OFFLINE / NEXT_POLL_TIME / TIMEOUT_COUNT）。
2. 通訊驅動層 (Modbus Driver)
 * 職責：與實體硬體（RS485/TCP）打交道的唯一出入口。
 * 重點要求：
   * 絕對單一實例 (Singleton)：整個程式只能有一個實體持有 Serial Port 或 Socket，保證物理層的「唯一聲音」。
   * 不處理任何業務邏輯，只負責純粹的位元組收發（Raw Bytes I/O）。
   * 嚴格執行 Timeout 判定並向上層拋出標準化例外（Exception），不負責重試（重試交由排程器或 Adapter 決定）。
3. 設備解析器 (Device Adapters)
 * 職責：扮演翻譯官，將原始二進位資料轉換為具備業務意義的數值（或反向封裝）。
 * 重點要求：
   * 實作工廠模式 (Factory Pattern)，區分 StandardAdapter 與多種 CustomAdapter（非標設備）。
   * 讀取翻譯：根據地圖檔，將 Driver 收到的 Hex Data 拆解、乘上 Scale（如 0.1）、轉換大小端、組裝成 JSON 格式。
   * 寫入翻譯：將上層傳來的目標數值，反向轉換成 Modbus 允許的 Register 格式。
4. 總線排程引擎 (Bus Master Scheduler)
 * 職責：系統的心臟，負責任務調度與總線鎖的管理。
 * 重點要求：
   * 快慢車道管理：維護一個 PriorityQueue（快車道寫入）與一個基於時間戳的 Min-Heap（慢車道輪詢）。
   * 原子化寫入流程 (上鎖機制)：接手快車道任務 -> 鎖定總線 -> 寫入 -> 成功則立即觸發讀取 -> 放鎖。
   * 防餓死機制：強制計數快車道連續執行次數（上限 5 次），超過即強制讓出 CPU 執行一次慢車道。
   * 黑名單/退避機制：攔截 Driver 拋出的 Timeout 例外，累計失敗次數。超過閾值則通知「狀態中心」將設備標記為 OFFLINE，並將下一次輪詢時間推遲（例如 60 秒後探測）。
5. 外部通訊代理 (MQTT Agent)
 * 職責：與外界（MQTT Broker, HA）互動的橋樑。
 * 重點要求：
   * 狀態發佈 (Publisher)：接收排程器完成的資料，發佈至對應的 telemetry topic。
   * 指令訂閱 (Subscriber)：監聽控制 Topic。收到外部寫入指令後，進行基礎格式校驗，然後推入 Bus Master 的快車道佇列（不直接操作 Driver）。
   * HA Discovery：僅在程式啟動連線成功，或設備從 OFFLINE 轉 ONLINE 的當下，依據地圖檔動態生成並發送 Home Assistant Config Payload。
二、 資料與控制流向 (Data & Control Flow)
以下是系統運行時的三種核心情境流向，請確認這是否符合你的底層預期：
1. 慢車道：常規輪詢流向 (Read Polling)
 * [Bus Master] 檢查時間，發現 UID 2 已達輪詢間隔 (10s)。
 * [Bus Master] 檢查快車道佇列是否為空 -> 是，則繼續。
 * [Bus Master] 向 [Device Adapter (UID2)] 請求建立讀取指令 (Payload)。
 * [Bus Master] 將 Payload 丟給 [Modbus Driver] 發送。
 * [Modbus Driver] 收到實體層回應，回傳 Hex Data 給 [Bus Master]。
 * [Bus Master] 將 Hex Data 交給 [Device Adapter (UID2)] 解析成 JSON 數據。
 * [Bus Master] 將 JSON 數據丟給 [MQTT Agent] 發佈。
 * [Bus Master] 更新 UID 2 的下次輪詢時間 (CurrentTime + 10s)，完成讓道。
2. 快車道：控制寫入流向 (Atomic Write)
 * [MQTT Agent] 收到外部針對 UID 4 的寫入指令。
 * [MQTT Agent] 將指令推入 [Bus Master] 的快車道佇列。
 * [Bus Master] 當前正在輪詢 UID 1，等待 UID 1 流程結束。
 * [Bus Master] 偵測到快車道有任務，啟動總線上鎖。
 * [Bus Master] 要求 [Device Adapter (UID4)] 建立寫入 Payload。
 * [Bus Master] 呼叫 [Modbus Driver] 執行寫入 (失敗則依設定重試最高 3 次)。
 * (寫入成功後) [Bus Master] 不放鎖，立刻建立 UID 4 的讀取 Payload。
 * [Bus Master] 呼叫 [Modbus Driver] 讀取 UID 4 當前最新狀態。
 * [Bus Master] 將讀回狀態經 Adapter 解析後，交由 [MQTT Agent] 發佈（完成狀態同步）。
 * [Bus Master] 解除總線鎖，檢查連續快車道執行次數。未達上限則繼續消化快車道；若達上限，則強制執行一次慢車道。
3. 防禦機制：設備離線流向 (Timeout Handling)
 * [Bus Master] 輪詢 UID 3，[Modbus Driver] 拋出 TimeoutException。
 * [Bus Master] 攔截例外，UID 3 的 TIMEOUT_COUNT +1。
 * 若 TIMEOUT_COUNT 達 5 次：
   * [Bus Master] 通知 [Config & State Registry] 將 UID 3 設為 OFFLINE。
   * [Bus Master] 將 UID 3 的下次輪詢時間設為 CurrentTime + 60s (進入慢速探測模式)。
   * [MQTT Agent] 發佈 UID 3 狀態為 unavailable，讓 HA 介面反灰。
